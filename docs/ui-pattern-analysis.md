# UI Pattern Analysis: AI Education & Student Records

This report analyzes core UI/UX patterns captured from 9 competitors to inform the high-fidelity design of Uni Folia.

---

## 1. Hero & Messaging Patterns
### [Pattern] The "Evidence-First" Headline
- **Observed in**: VibeOn, UniGo, AIBU.
- **Copy Structure**: `[Data/Record]로 [Future Result]를 [Action Verb]`
  - 예: "120만 데이터로 합격을 진단하세요."
- **Why it works**: 입시의 불확실성을 '데이터'라는 객관적 지표로 해소해 준다는 인상을 줌.
- **Uni Folia Reinterpretation**: "실제 기록으로 안전한 탐구와 드래프팅을 시작하세요."
  - (차별화: 단순 확률 예측이 아닌 '기록 기반의 정직함' 강조)

### [Pattern] Visual Value Anchor (Preview)
- **Observed in**: HakZzongPro, VibeOn.
- **Visual Structure**: Hero 옆에 실제 분석 리포트 PDF나 대시보드 화면을 띄워 서비스의 결과물을 즉시 보여줌.
- **Why it works**: 가입 전 서비스의 '실체'를 확인하게 하여 전환율(Conversion)을 높임.
- **Uni Folia Reinterpretation**: 워크숍의 결과물인 '기록 기반 탐구 로드맵'을 시각화하여 배치.

## 2. Feature & Workflow Patterns
### [Pattern] Step-by-Step Value Delivery
- **Observed in**: Hamaroom, VibeOn.
- **Structure**: `1. 입력 -> 2. 분석 -> 3. 결과` 식의 가로/세로 카드 리스트.
- **Why it works**: 복잡한 인공지능 프로세스를 사용자가 통제 가능한 '단계'로 나누어 심리적 부담을 낮춤.
- **Uni Folia Reinterpretation**: `목표 설정 -> 생기부 정리 -> AI 워크숍 -> 안전한 드래프팅`의 4단계 워크플로우를 타사보다 더 강조하여 '과정의 전문성' 포지셔닝.

### [Pattern] The "Safe Generation" Guard
- **Observed in**: Hamaroom (교사용).
- **Structure**: 문장을 생성해주되, 근거가 되는 키워드를 사용자가 직접 선택하도록 유도.
- **Why it works**: '대필'이 아닌 '보조'임을 명확히 하여 교육적/윤리적 신뢰를 확보.
- **Uni Folia Reinterpretation**: "기록이 부족하면 생성하지 않습니다"라는 경고 패턴을 UI 곳곳에 배치하여 독보적인 신뢰도 구축.

## 3. Trust & Social Proof Patterns
### [Pattern] Status & Verification Badges
- **Observed in**: VibeOn, RiroSchool.
- **Structure**: "교육부 가이드 준수", "ISO 인증", "X개 학교 도입" 로고.
- **Why it works**: 국가기관이나 다수의 사용자가 검증했다는 느낌을 줌.
- **Uni Folia Reinterpretation**: "허위 미화 금지", "기록 기반 원칭 준수" 등 우리만의 **'Safety Badge'**를 제작하여 Hero 하단에 배치.

## 4. CTA & Lead Generation Patterns
### [Pattern] The "Curiosity" Lead
- **Observed in**: UniGo, AIBU.
- **Structure**: `[내 점수/전공적합성 확인하기]`와 같이 실시간 결과가 궁금하게 만드는 문구.
- **Why it works**: '시작하기'보다 더 구체적인 이득(Benefit)을 제시.
- **Uni Folia Reinterpretation**: `[내 생기부의 핵심 역량 진단하기]` 또는 `[나만의 탐구 퀘스트 받기]`.

---

## 5. Mobile vs Desktop Structural Changes
- **Observation**: 대다수 사이트가 모바일에서 `GNB`를 `Hamburger Menu`로 숨기고, 가로형 피처 카드를 세로형 스태킹(Stacking)으로 전환함.
- **Uni Folia Strategy**: 워크숍 기능이 핵심이므로, 모바일에서는 메뉴는 최소화하되 현재 진행 중인 '퀘스트'나 '드래프트' 카드를 최상단에 고정하는 **'진행 중심(Progress-focused)'** 레이아웃 도입.

---

## 6. Strategic Conclusions for Uni Folia
- **차별점 극대화**: 타사가 '결과(PDF)'를 강조할 때, 우리는 **'작업실(Drafting Space)'**의 편리함과 안전함을 UI로 시연해야 함.
- **버려야 할 패턴**: 타사가 흔히 사용하는 **'화려한 전광판 효과(누구님이 합격했습니다 등)'**는 우리 브랜드의 진정성(Evidence-grounded)을 해칠 수 있으므로 지양하고, 차분하고 정돈된 **'전문가 툴'** 느낌의 디자인 유지.
