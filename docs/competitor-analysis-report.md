# 2026 AI-Driven University Admission Competitor Analysis Report

This report provides a structural analysis of 9 key competitors to Uni Folia, focusing on UI/UX patterns, conversion flows, and trust mechanisms.

---

## 1. Individual Site Analysis (15-Item Structural Analysis)

### 1️⃣ 바이브온 (VibeOn) - https://www.vibeon.ai/
1. **한 줄 정의**: 누적 합격 데이터를 바탕으로 생기부를 진단하고 합격 가능성을 예측하는 B2C 전문 AI 진단 플랫폼.
2. **타겟 사용자**: 수시 전형을 준비하는 고교생 및 학부모.
3. **첫 화면 핵심 메시지**: "데이터로 증명하는 나의 합격 확률, 바이브온 AI 진단."
4. **CTA 구조**: `[지금 무료 진단하기]` (메인), `[샘플 리포트 보기]` (보조).
5. **정보구조**: 상단 GNB: 생기부 진단, 대학/전형 분석, 활동 설계, 커뮤니티. 푸터: 회사 정보, CS, 정책.
6. **기능 소개 방식**: 스텝별(Step 1~3) 카드 UI와 화려한 데이터 시각화 샘플 노출.
7. **FAQ 유무**: 중급 품질. 결제/진단 방식 위주의 정적 FAQ 페이지 운영.
8. **문의/협업문의 구조**: 우측 하단 카카오톡 채널 플로팅 버튼. 기업용 도입 문의 별도 폼 제공.
9. **회원가입/로그인 유도**: "첫 진단 무료"를 강조하며 카카오/네이버 1초 간편 가입 유도.
10. **신뢰 구축 장치**: "120만 건 이상의 누적 데이터", "현직 입시 전문가 검수" 문구 및 언론 보도 로고.
11. **UI 강점**: 신뢰감을 주는 딥블루(Navy) 톤앤매너, 가독성 높은 그리드 레이아웃, 인포그래픽 중심.
12. **UX 강점**: 복잡한 데이터를 수치화하여 직관적인 '성적표' 형태로 제공, 명확한 가입 혜택(무료).
13. **UI/UX 약점**: 메뉴가 많아 초기 진입 시 다소 복잡해 보임, 모바일에서 일부 테이블 UI 가독성 저하.
14. **참고 요소**: 대시보드 형태의 분석 결과 리포트 디자인, 카카오톡 기반 알림톡 연동.
15. **피해야 할 요소**: 사용자의 능동적 기록보다는 단순 '예측 결과'에 치중하여 자칫 수동적인 느낌을 줄 수 있음.

### 2️⃣ 유니고 (UniGo) - https://unigo.kr/
1. **한 줄 정의**: 생기부 데이터 매칭을 통해 지원 가능한 대학/학과를 추천해주는 실전 지원 전략 솔루션.
2. **타겟 사용자**: 수시 지원 대상 고3 학생 및 재수생.
3. **첫 화면 핵심 메시지**: "나의 생기부에 가장 잘 어울리는 대학은 어디일까?"
4. **CTA 구조**: `[나에게 맞는 전형 찾기]` (메인).
5. **정보구조**: 매칭 진단, 생기부 분석, 대입 가이드 중심의 심플한 구조.
6. **기능 소개 방식**: "나의 데이터 vs 대학 기준" 비교 대조표 UI 활용.
7. **FAQ 유무**: 보통 품질. FAQ 리스트 및 검색 기능 제공.
8. **문의/협업문의 구조**: 하단 고객센터 정보 노출 및 1:1 웹 문의 폼.
9. **회원가입/로그인 유도**: 진단 마지막 단계에서 결과 확인을 위해 회원가입 필수 유도.
10. **신뢰 구축 장치**: 대학별 최신 모집 요강 반영 공지 및 매칭 정확성 강조.
11. **UI 강점**: 깔끔한 화이트 톤, 중요 강조 사항에 라임/오렌지색 포인트 컬러 사용.
12. **UX 강점**: '매칭'이라는 목적에 집중한 간결한 흐름.
13. **UI/UX 약점**: 초기 로딩 속도가 다소 느림, 콘텐츠 양에 비해 여백이 다소 과한 구간 있음.
14. **참고 요소**: 개인화된 대학 추천 카드 UI.
15. **피해야 할 요소**: 분석이 끝난 후 '유료 결제' 장벽이 나타날 때의 급격한 이탈감.

2. **타겟 사용자**: 입시 학원 운영자, 진학 상담 교사, 개인 컨설턴트.

4. **CTA 구조**: `[무료 체험 신청하기]` (메인), `[도입 문의]` (보조).
5. **정보구조**: 서비스 소개, 주요 기능, 도입 사례, 가격 안내.
6. **기능 소개 방식**: 세특 자동 완성, 대학 배치표 등 '업무 효율성' 강조 위주.
7. **FAQ 유무**: 하급 품질. 랜딩 페이지 하단 요약 FAQ만 존재.
8. **문의/협업문의 구조**: 전문적인 B2B 상담 신청 폼(이름, 학원명, 연락처 등).
9. **회원가입/로그인 유도**: 일반 가입보다는 기업용 승인 가입 위주.
10. **신뢰 구축 장치**: 제휴 학원 수, 성공 입시 사례 수치 강조.
11. **UI 강점**: 관리자 툴(SaaS) 느낌의 정돈된 인터페이스, 기능 단위의 아이콘 강조.
12. **UX 강점**: 전문가가 사용하기에 필요한 고급 데이터(배치표 등)를 전면에 배치.
13. **UI/UX 약점**: 일반 학생/학부모가 보기에는 다소 딱딱하고 사무적인 분위기.
14. **참고 요소**: 다수의 학생 생기부를 관리하는 '워크스페이스' 개념 UI.
15. **피해야 할 요소**: 생성된 결과물의 '복사-붙여넣기'식 느낌(인위적인 문장 생성이 강조됨).

## 4. Uni Folia 전략적 제안

### ✅ Uni Folia에 바로 반영할 수 있는 점
- **진단 리포트 이미지화**: 현재의 채팅/워크숍 과정이 얼마나 훌륭한 결과(PDF/리포트)로 이어지는지 랜딩 페이지에서 시각적 샘플로 보여줘야 함.
- **카톡 채널 연동**: 문의 접근성을 높이기 위해 카카오톡 상담 채널을 플로팅 버튼으로 추가.
- **온보딩 마법사**: 첫 진입 시 목표 대학/관심 분야를 선택하는 3~4단계 위저드 도입으로 개인화 강화.

### ❌ Uni Folia의 차별점을 위해 버려야 할 요소
- **단순 예측(Probability)**: 타사가 집중하는 "합격 확률 00%" 식의 불안 마케팅을 지양하고, "무엇을 더 해야 하는가"에 집중하는 가이드 기능에 자원 집중.
- **대량 자동 생성**: 하마룸처럼 한꺼번에 문장을 만들어내는 방식은 학생의 성장을 저해함. "기록 확인 -> 질문 -> 문장 제안 -> 학생 수정"의 **실행형 워크플로우** 고수.
- **포털형 메뉴**: 리로스쿨처럼 모든 기능을 다 보여주지 말고, 현재 학생의 진행 단계(Phase)에 필요한 기능만 노출하는 **집중형(Focused) UI** 유지.
